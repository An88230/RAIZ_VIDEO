# Creative OS Local Agent Architecture

Phase 37 defines architecture and contracts only. It does not build the Local
Agent server, execute commands, add desktop automation, or change RAIZ_VIDEO
render behavior.

## Decision

Creative OS is the right-hand command layer for Nabil. It receives natural
commands, converts them into structured command JSON, and sends those commands to
a future local agent. The local agent may control RAIZ_VIDEO only through
explicitly registered allowed actions.

`hectorg2211/jarvis` is a conceptual reference only for wake trigger,
microphone input, local action routing, and assistant flow. No code is copied.

## Components

```text
Creative OS UI
  -> Voice Input Boundary
  -> Intent Router
  -> creative_os_command.json
  -> Local Agent Server
  -> Allowed Action Registry
  -> RAIZ_VIDEO Action Adapter
  -> Gemini TTS Adapter
  -> Audit Log Boundary
  -> creative_os_action_result.json
  -> Voice Output Boundary
  -> Creative OS UI
```

### Creative OS UI

The UI remains separate from RAIZ_VIDEO. It may capture speech, display plans,
request confirmation, and play live assistant speech. It does not store
production API keys and does not execute system commands.

### Voice Input Boundary

Voice input is converted to text before action routing. Wake trigger and
microphone handling belong to Creative OS or a future Local Agent shell, not to
RAIZ_VIDEO core.

### Intent Router

The router turns natural language into `creative_os_command.schema.json`. It must
choose a single `requested_action`, fill a structured payload, and include safety
metadata. It does not perform side effects.

### Local Agent Server

Future local-only service. It receives `creative_os_command.json`, validates it,
checks the Allowed Action Registry, asks for confirmation when required, runs the
registered adapter, writes an audit entry, and returns
`creative_os_action_result.json`.

The server is not implemented in Phase 37.

### Allowed Action Registry

The registry is the only execution surface. Every action must define:

- `action_id`
- input schema
- output schema
- allowed paths
- side effects
- confirmation requirement
- audit log entry

No free-form terminal command may be routed from model output.

### RAIZ_VIDEO Action Adapter

This future adapter maps allowed action IDs to existing RAIZ_VIDEO workflows, for
example converting a creative brief, running the guarded local render path, or
inspecting artifacts. It must call known scripts or local API endpoints by fixed
contract, not arbitrary shell text.

### Gemini TTS Adapter

Gemini TTS is the official RAIZ voice layer. Creative OS may use Gemini TTS for
live UI speech. RAIZ_VIDEO render voice must be generated by the future Local
Agent as a saved asset under:

```text
storage/jobs/{job_id}/assets/voice/
```

The browser never receives the Gemini API key in the production path.

### Voice Output Boundary

Live UI speech and render voice are separate. Live UI speech can respond quickly
to Nabil. Render voice is a file artifact that must be reproducible and auditable.

### .env Key Boundary

Production Gemini credentials live only in local `.env` consumed by the Local
Agent. Creative OS sends commands, not secrets.

### Audit Log Boundary

Every action result must be auditable. The future Local Agent should write an
append-only local audit entry with command ID, action ID, confirmation state,
artifacts, errors, and timestamps.

## First Flow

1. Nabil says: "يا رايز، حوّل نص الانطفاء اللامع إلى فيديو 30 ثانية..."
2. Creative OS produces `creative_os_command.json`.
3. Local Agent validates the command.
4. Registry allows `run_local_render` only if prerequisites are present and
   confirmation is satisfied.
5. RAIZ_VIDEO Action Adapter performs fixed, known local steps.
6. Gemini TTS Adapter generates a render voice asset if needed.
7. Local Agent returns `creative_os_action_result.json`.

## Non-goals

- No Local Agent implementation in Phase 37.
- No arbitrary shell execution.
- No desktop automation.
- No ElevenLabs.
- No YouTube, Google Drive, n8n, or publishing automation.
- No Remotion template changes.
- No `vendor/` changes.
